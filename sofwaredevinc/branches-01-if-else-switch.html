if / else / switch

Programs rarely execute every line in order. Branching lets the program choose which code to run based on conditions evaluated at runtime. This is how C makes decisions.

The Big Picture

Without branching, a program is just a recipe — it executes the same steps in the same order every time regardless of what the input is. Branching gives a program the ability to respond to its situation: different input, different data, different results — different path through the code.

The condition is evaluated at runtime. Exactly one branch executes. Both paths rejoin after the if/else.

Boolean Values in C

C has no dedicated boolean type in its original form. Every condition is just an integer expression. The rule is simple: zero is false, everything else is true. This is the foundation of every if statement in C.

if (0)   { ... }  // never executes — 0 is false
if (1)   { ... }  // always executes — 1 is true
if (-42) { ... }  // always executes — any non-zero is true
if (ptr) { ... }  // executes if ptr is not NULL (NULL == 0)

C99 added <stdbool.h> which defines bool, true (1), and false (0). These are aliases — the underlying integer rules never change. You'll see both styles in real code.

Short-circuit evaluation
&& stops evaluating as soon as it finds a false term. || stops as soon as it finds a true term. This is not just an optimization — it's a guarantee you can rely on. The pattern if (ptr != NULL && ptr->value > 0) is safe because the second condition only runs when ptr is non-NULL.

if / else if / else

if (score >= 90) {
    printf("A\n");
} else if (score >= 80) {
    printf("B\n");
} else if (score >= 70) {
    printf("C\n");
} else {
    printf("F\n");
}

The conditions are tested in order. The first branch whose condition is true executes, then the entire chain is skipped. The else at the end is a catch-all — it runs only if nothing above matched. You can have an if with no else, in which case the program simply does nothing if the condition is false.

An else-if chain tests conditions in order and executes exactly one branch. Once a condition matches, the rest are skipped entirely.

The Ternary Operator

For simple two-outcome decisions, the ternary operator collapses an if/else into a single expression. Use it when each branch produces a value — not for complex logic.

// if/else form:
int max;
if (a > b) max = a;
else       max = b;

// ternary form — identical behavior:
int max = (a > b) ? a : b;

// useful inline:
printf("Score: %d (%s)\n", score, score >= 60 ? "pass" : "fail");

switch

A switch statement compares one integer expression against a series of constant values. It's cleaner than a long else-if chain when you're dispatching on a single variable that can take several known values.

switch (command) {
    case 'q':
        printf("Quitting\n");
        break;
    case 'h':
        print_help();
        break;
    case 'r':
    case 'R':                    // two cases, same body — intentional fall-through
        reload();
        break;
    default:
        printf("Unknown command\n");
        break;
}

Fall-Through: Intentional vs. Accidental

Unlike if/else, switch does not automatically stop at the end of a matching case. Without a break, execution falls through into the next case. This is sometimes useful (stacking cases for the same body) but is one of the most common sources of bugs in C.

break exits the switch immediately. Without it, execution continues into the next case regardless of whether its label matches.
Always put break at the end of every case unless fall-through is intentional and clearly commented. Accidental fall-through is a silent bug — the compiler will not warn you, and the code will appear to work until it doesn't.

switch vs. if/else: When to Use Which

Use switch whenUse if/else when

switch cannot test strings
A common beginner mistake is writing switch (str) to compare strings. This compares the pointer address, not the string content. For string dispatch, use strcmp() in an if/else chain instead.

Common Problems

Pitfall 01
= Instead of ==
Writing if (x = 5) instead of if (x == 5). The assignment sets x to 5 and returns 5, which is always true. This is valid C — the compiler often won't warn. The condition never behaves as intended. Enable -Wall and always use == for comparisons.
Pitfall 02
Missing break in switch
The most common switch bug. Execution falls silently into the next case. The fix is simple — add break. If fall-through is intentional, add a comment: /* fall through */. Some compilers warn about unlabelled fall-through with -Wimplicit-fallthrough.
Pitfall 03
Dangling else
An else always binds to the nearest preceding if, not the one you might visually intend. Misleading indentation can hide this. Always use braces {} even for single-statement bodies to make the structure unambiguous.
Pitfall 04
Comparing Floats with ==
Floating-point arithmetic introduces tiny rounding errors. 0.1 + 0.2 == 0.3 is false in C. Instead compare with a tolerance: fabs(a - b) < 1e-9. Never use == or != with float or double.
Pitfall 05
Omitting Braces on Single-Line Bodies
if (x) foo(); bar(); — bar() always executes regardless of the condition. Only foo() is inside the if. A second line added below a brace-less if is never part of it. Use braces, always.
Pitfall 06
Checking Order in else-if Chains
If a broader condition appears before a narrower one, the narrower case can never be reached. Example: putting score >= 60 before score >= 90 — every A student gets labelled as a C. Order your conditions from most specific to least.

The Dangling Else, Illustrated

// What you wrote:
if (x > 0)
    if (x > 100)
        printf("big\n");
else
    printf("negative\n");  // ← you think this is the else for x > 0...

// What C sees (else binds to nearest if):
if (x > 0) {
    if (x > 100)
        printf("big\n");
    else
        printf("negative\n");  // ...but it's actually the else for x > 100
}

// Fix: use braces to make intent explicit
if (x > 0) {
    if (x > 100)
        printf("big\n");
} else {
    printf("negative\n");   // now clearly the else for x > 0
}
Takeaway
Branching is how programs make decisions. In C, any non-zero value is true. Use == not = in conditions. Always put break in switch cases unless fall-through is intentional and documented. Always use braces, even for one-line bodies. Test else-if chains from most specific to least specific. Never compare floats with ==.
