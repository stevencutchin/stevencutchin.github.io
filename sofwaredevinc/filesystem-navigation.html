Navigating the
File System

C gives you direct access to the OS file system through a small set of POSIX headers. Learn to walk directory trees, inspect file metadata, and build tools that reason about paths.

The Headers You'll Use

Three headers cover the overwhelming majority of filesystem work in C. Each maps to a different concern: where am I?, what entries are here?, and what kind of thing is this entry?

<unistd.h> <dirent.h> <sys/stat.h>

The file system is a tree — directories are nodes, files are leaves. Every path is a route from root to a node.

The OS represents this tree using inodes — internal data structures that store everything about a file except its name. The name lives in the directory entry that points to the inode. This is why you can have multiple names (hard links) pointing to the same underlying file.

Knowing Where You Are — <unistd.h>

Before you can navigate anywhere, you need to know your current position in the tree. Every process has a current working directory — the implicit base for any relative path you use.

getcwd()
char *getcwd(char *buf, size_t size);
Fills buf with the absolute path of the current working directory. size must be large enough to hold the path.
returns buf on success
returns NULL on failure, sets errno
chdir()
int chdir(const char *path);
Changes the process's current working directory to path. Both absolute (/home/user) and relative (../sibling) paths work.
returns 0 on success
returns -1 on failure, sets errno
#include <unistd.h>
#include <stdio.h>

int main(void) {
    char cwd[1024];

    if (getcwd(cwd, sizeof(cwd)) == NULL) {
        perror("getcwd");
        return 1;
    }
    printf("Current dir: %s\n", cwd);

    if (chdir("..") == -1) {
        perror("chdir");
        return 1;
    }
    getcwd(cwd, sizeof(cwd));
    printf("Now in:      %s\n", cwd);

    return 0;
}
Key Concept
chdir() only affects the current process. It does not change the shell that launched your program. This is why you cannot write a C program that changes your terminal's directory — that shell is a separate process.

Reading a Directory — <dirent.h>

To see what's inside a directory you follow a three-step ritual: open the directory stream, read entries one at a time in a loop, then close the stream. This mirrors exactly how you handle a file with fopen/fread/fclose.

opendir()
DIR *opendir(const char *name);
Opens a directory stream. Returns an opaque DIR* handle — you never look inside it directly, you just pass it to other functions.
returns DIR* on success
returns NULL on failure
readdir()
struct dirent *readdir(DIR *dirp);
Returns a pointer to the next directory entry. Call repeatedly in a loop. The pointer is owned by the library — do not free it.
returns struct dirent* or NULL at end
closedir()
int closedir(DIR *dirp);
Closes the directory stream and frees its resources. Always call this — a DIR* is a resource just like a FILE*.
returns 0 on success
returns -1 on failure
struct dirent
d_name[] — entry name (char array)
d_type — entry type constant
Each call to readdir() fills this struct. The two fields you'll use most are the entry's name and its type tag.

The opendir / readdir / closedir loop — readdir() returns NULL to signal end-of-directory, not an error.

The d_type Field

Each struct dirent carries a d_type field that tells you what kind of filesystem object the entry is. The two you'll use constantly are:

DT_REG   // regular file
DT_DIR   // directory
DT_LNK   // symbolic link
DT_UNKNOWN // type not available — fall back to stat()
⚠ d_type is not always populated. Some filesystems (notably older Linux ext2 volumes) return DT_UNKNOWN for every entry. If your code must be portable, always be prepared to fall back to stat() for type information.

A complete directory listing

#include <stdio.h>
#include <dirent.h>
#include <string.h>

int main(int argc, char *argv[]) {
    const char *path = (argc > 1) ? argv[1] : ".";

    DIR *dir = opendir(path);
    if (dir == NULL) {
        perror(path);
        return 1;
    }

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        // skip hidden entries (. and ..)
        if (entry->d_name[0] == '.') continue;

        char marker = (entry->d_type == DT_DIR) ? '/' : ' ';
        printf("%c  %s\n", marker, entry->d_name);
    }

    closedir(dir);
    return 0;
}

Inspecting File Metadata — <sys/stat.h>

A directory entry only tells you a file's name and rough type. For everything else — size, permissions, timestamps, the precise inode type — you call stat(). It fills a struct stat with a full profile of the file.

stat()
int stat(const char *path, struct stat *buf);
Follows symbolic links and reports on the target. Use this for almost everything.
returns 0 on success
returns -1, sets errno
lstat()
int lstat(const char *path, struct stat *buf);
Does not follow symbolic links — reports on the symlink itself. Use when you want to detect links rather than transparently traverse them.
returns 0 on success
returns -1, sets errno

struct stat — the complete metadata profile returned by stat() and lstat()

The st_mode field and S_ macros

The st_mode field packs file type and permission bits together into a single integer. Never test it directly — use the provided macros:

S_ISREG(m)   // true if regular file
S_ISDIR(m)   // true if directory
S_ISLNK(m)   // true if symbolic link

// permissions are the low 9 bits of st_mode
// e.g.  st.st_mode & 0777  gives rwxrwxrwx mask

Putting stat and dirent together

#include <stdio.h>
#include <dirent.h>
#include <sys/stat.h>
#include <string.h>

void list_dir(const char *path) {
    DIR *dir = opendir(path);
    if (!dir) { perror(path); return; }

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        if (entry->d_name[0] == '.') continue;

        // build the full path for stat()
        char fullpath[2048];
        snprintf(fullpath, sizeof(fullpath), "%s/%s", path, entry->d_name);

        struct stat st;
        if (stat(fullpath, &st) == -1) {
            perror(fullpath);
            continue;
        }

        if (S_ISDIR(st.st_mode))
            printf("DIR   %s/\n", entry->d_name);
        else if (S_ISREG(st.st_mode))
            printf("FILE  %s  (%lld bytes)\n",
                   entry->d_name, (long long)st.st_size);
    }

    closedir(dir);
}

int main(int argc, char *argv[]) {
    list_dir(argc > 1 ? argv[1] : ".");
    return 0;
}

Recursive Directory Walk

Because the filesystem is a tree, the natural way to process it fully is recursion. When you encounter a DT_DIR entry (or confirmed directory via stat()), you call your listing function again on that subdirectory.

void walk(const char *path, int depth) {
    DIR *dir = opendir(path);
    if (!dir) { perror(path); return; }

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        if (entry->d_name[0] == '.') continue;

        // indent by depth
        for (int i = 0; i < depth; i++) printf("  ");
        printf("%s\n", entry->d_name);

        if (entry->d_type == DT_DIR) {
            char child[2048];
            snprintf(child, sizeof(child), "%s/%s", path, entry->d_name);
            walk(child, depth + 1);   // recurse
        }
    }

    closedir(dir);
}
⚠ Watch out for symbolic link cycles. If a symlink points to an ancestor directory, a naive recursive walk will loop forever. Use lstat() instead of checking d_type, so you visit the symlink itself rather than following it.
Mental Model
opendir()/readdir()/closedir() are the directory-level equivalent of fopen()/fread()/fclose(). Both pairs follow the same resource lifecycle: open → use in a loop → close. Always close what you open.

Compiling

All three headers are part of POSIX and ship with every Linux/macOS toolchain. No extra linker flags are needed:

gcc -Wall -Wextra -o myls myls.c

On Linux you may additionally want -D_DEFAULT_SOURCE or -D_XOPEN_SOURCE=700 if your compiler warns about implicit function declarations for POSIX functions — these flags tell the compiler to expose the full POSIX symbol set.
