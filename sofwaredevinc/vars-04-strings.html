Strings

C has no string type. A string is a char array with a zero byte at the end. That single design decision — the null terminator — is the source of both C's string flexibility and most of its string bugs.

What a String Is

In C, a string is a sequence of char values stored in contiguous memory, followed by a '\0' (null terminator) — the byte with value zero. The null terminator is not part of the string's content; it's a sentinel that tells every string function where the string ends. The language provides no built-in string type — strings are just arrays of bytes with a convention about how they end.

"Hello" occupies 6 bytes: 5 for content, 1 for the null terminator. strlen counts content bytes only. The array must hold both.

Declaring Strings

There are two main ways to create a string in C — as a stack-allocated character array or as a pointer to a string literal. They look similar but behave very differently.

// Array form — modifiable, stored on the stack
char name[64];                  // uninitialized buffer, room for 63 chars + '\0'
char greeting[] = "Hello";      // compiler sizes it: 6 bytes (5 + '\0')
char buf[64]   = "Hello";      // 64-byte buffer, first 6 bytes set

greeting[0] = 'h';              // OK — array is modifiable

// Pointer form — points to read-only string literal
const char *msg = "Hello";      // msg points to static read-only memory

msg[0] = 'h';                   // UNDEFINED BEHAVIOR — may crash
Array form owns its own copy of the bytes. Pointer form points into read-only program memory. Never write through a pointer to a string literal.

Essential String Functions

C provides string operations through <string.h>. These functions all rely on the null terminator to find the end of the string — which means they all break silently if the null terminator is missing or the buffer is too small.

#include <string.h>

char buf[64];
char *src = "Hello, world";

// length
size_t len = strlen(src);         // 12 — does not count '\0'

// safe copy
strncpy(buf, src, sizeof(buf) - 1);
buf[sizeof(buf) - 1] = '\0';      // always ensure termination

// comparison — result is 0 when equal
if (strcmp(buf, src) == 0)
    printf("strings are equal\n");

// safe format
snprintf(buf, sizeof(buf), "Score: %d", 42);

Strings Are Pointers

When you pass a string to a function, you're passing a char * — a pointer to the first character. The function receives the address of the first byte and reads forward until it hits '\0'. There is no size information in the pointer itself. This is why every function that writes to a string needs a separate size argument.

// These two declarations are equivalent for function parameters
void print_str(char *s)    { while (*s) putchar(*s++); }
void print_str(char  s[]) { while (*s) putchar(*s++); }

// Walking a string with a pointer
char *p = "Hello";
while (*p != '\0') {
    printf("%c\n", *p);
    p++;
}
String traversal with a pointer. *p reads the current character, p++ moves to the next. '\0' (value 0) is falsy, ending the loop.

Buffer Size: The Critical Rule

The single most important rule in C string handling: every buffer that holds a string must have at least one extra byte for the null terminator. A string of length N requires an array of size N+1. Getting this wrong is the root cause of buffer overflows — one of the most exploited vulnerability classes in C programs.

// String "Hello" has length 5 — needs 6 bytes minimum
char buf[5] = "Hello";    // WRONG — no room for '\0', overflow
char buf[6] = "Hello";    // correct

// When sizing buffers for user input, be generous
char name[256];           // 255 usable chars + '\0'
fgets(name, sizeof(name), stdin);  // reads at most 255 chars + '\0'

// snprintf respects your buffer limit — always use over sprintf
char out[32];
snprintf(out, sizeof(out), "Hi %s", name); // safe: truncates if necessary

Common Problems

Pitfall 01
Buffer Too Small for Null Terminator
Declaring char buf[5] = "Hello" has no room for '\0'. The string overwrites adjacent memory. Every string buffer must be at least length + 1. When in doubt, make the buffer larger than you think you need.
Pitfall 02
Comparing Strings with ==
str1 == str2 compares pointer addresses, not string contents. Two different arrays containing "hello" will compare as unequal because they live at different addresses. Always use strcmp(str1, str2) == 0 to compare string content.
Pitfall 03
Using strcpy/strcat Without Size
strcpy(dst, src) copies until it finds '\0' in src, with no check that dst is large enough. If src is longer than dst's buffer, the copy overflows into adjacent memory. Always use strncpy or better yet snprintf with an explicit size limit.
Pitfall 04
strncpy Doesn't Always Null-Terminate
If the source string is longer than n, strncpy copies n bytes but adds no '\0'. The result is an unterminated array. Any subsequent string operation on it will read past the end. Always explicitly set buf[n-1] = '\0' after strncpy.
Pitfall 05
Modifying a String Literal
char *s = "hello"; s[0] = 'H'; is undefined behavior. String literals are stored in read-only memory. The program may crash with a segmentation fault, or silently corrupt other strings that share the same literal. Use the array form to get a modifiable copy.
Pitfall 06
Forgetting to Strip fgets Newline
fgets includes the '\n' in the buffer. strcmp(buf, "quit") will always fail because buf contains "quit\n". Strip it immediately: buf[strcspn(buf, "\n")] = '\0';.
Pitfall 07
Missing Null Terminator After Manual Construction
When building a string character by character, it's easy to forget to add '\0' at the end. Without it, every string function that reads the buffer will run past the end of your data into garbage memory — or crash. Always manually append '\0' when constructing strings byte by byte.
Pitfall 08
Confusing strlen and sizeof
strlen(s) counts characters up to but not including '\0' — the string's visible length. sizeof(s) returns the total array size in bytes, including the '\0' slot. On a pointer (not an array), sizeof returns the pointer size (typically 8), not the string length.
Takeaway
A C string is a null-terminated array of chars. Every string buffer must have one extra byte for '\0'. Compare strings with strcmp, never ==. Use snprintf over sprintf, strncpy over strcpy — and always verify null-termination after strncpy. Strip fgets's newline before comparing. Never write through a pointer to a string literal.
