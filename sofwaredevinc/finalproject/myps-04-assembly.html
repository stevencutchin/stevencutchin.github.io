Part 3 — Assembly, Sort, Display

An array of pointers, sorted

Take the dirent list from Part 2, build a ProcEntry for each PID, sort the array with a comparator chosen at runtime, print the column-aligned output, and free everything cleanly. The whole project lands here.

Allocate the ProcEntry array

You have n dirent pointers. You need an array of n ProcEntry pointers. Allocate that array first, before the loop that fills it:

ProcEntry **myprocs = malloc(sizeof(ProcEntry *) * n);
if (myprocs == NULL) {
    fprintf(stderr, "myps: malloc failed for ProcEntry array\n");
    /* still need to free dirents before exiting */
    for (int i = 0; i < n; i++) free(dirents[i]);
    free(dirents);
    return EXIT_FAILURE;
}
Three levels: stack pointer → array of heap pointers → ProcEntry structs (each owning two more heap strings).
Why array of pointers, not array of structs? Two reasons. First, the Create/Destroy contract returns and accepts pointers, not values. Second, sorting an array of pointers means swapping a couple of bytes per swap; sorting an array of structs means copying every field. Pointers are cheap, structs aren't.

The fill loop

For each dirent, build the path to its stat file, parse it, store the result. Some PIDs may have vanished between the scandir call and now (the /proc filesystem is volatile, remember from page 1) — handle NULL returns by storing NULL and skipping later:

int myprocs_count = 0;
for (int i = 0; i < n; i++) {
    char stat_path[PATH_MAX];
    snprintf(stat_path, sizeof(stat_path), "%s/%s/stat",
             base_path, dirents[i]->d_name);

    ProcEntry *entry = CreateProcEntryFromFile(stat_path);
    if (entry != NULL) {
        myprocs[myprocs_count++] = entry;
    }
    // silently skip missing/malformed files; the volatile nature of /proc
    // means a process may have exited between scandir and the open here.
}
You need limits.h PATH_MAX is declared in <limits.h>. Add #include <limits.h> to the top of myps.c. If you don't and PATH_MAX happens to be defined as a macro through some indirect include, your code will compile on your machine but fail on the grader's.

The qsort comparators

The project gives you pidSort verbatim. Note the double-pointer dance: qsort hands the comparator void * pointers to elements of the array. Our array elements are ProcEntry*, so the void* is a pointer to a pointer — and we cast and dereference it once to get the actual ProcEntry*:

static int pidSort(const void *aIn, const void *bIn)
{
    ProcEntry *f = *(ProcEntry **)aIn;        // cast then dereference
    ProcEntry *s = *(ProcEntry **)bIn;
    int rval = f->process_id - s->process_id;
    return rval;
}

static int commSort(const void *aIn, const void *bIn)
{
    ProcEntry *f = *(ProcEntry **)aIn;
    ProcEntry *s = *(ProcEntry **)bIn;
    return strcmp(f->comm, s->comm);
}
any array of any element type. It can't know the element type, so it hands you a pointer to an element. Your job is to cast and dereference. Element type here is ProcEntry* — so "pointer to element" is ProcEntry**, and one deref gets you the ProcEntry*.
Two casts, one dereference. Once you've seen this pattern, qsort is no longer mysterious.

Calling qsort with the chosen comparator

// Pick the comparator at runtime based on the flag.
int (*compare)(const void *, const void *) = sort_by_comm ? commSort : pidSort;

qsort(myprocs, myprocs_count, sizeof(ProcEntry *), compare);
Function pointers are values compare is a variable that holds a function pointer. The ternary picks one of two functions; the resulting pointer gets stored. qsort doesn't care which one — it just calls compare(...) on each pair it needs to compare. This is the same pattern as the strategy pattern in OOP languages.

Print the column header and the rows

The header line is part of the integration test. Copy this format string verbatim — don't reformat, don't change the column widths, don't add or remove spaces. Then loop through the array and print each entry, applying the -z filter at this stage:

fprintf(stdout,
        "%7s %7s %5s %5s %5s %7s %-25s %-20s\n",
        "PID", "PPID", "STATE", "UTIME", "STIME",
        "THREADS", "CMD", "STAT_FILE");

for (int i = 0; i < myprocs_count; i++) {
    if (zombies_only && myprocs[i]->state != 'Z') {
        continue;        // -z: skip non-zombies
    }
    PrintProcEntry(myprocs[i]);
}
The header format string is not flexible The widths and spacing must match PrintProcEntry's format string exactly so that columns line up and match the reference output. The two format strings above are identical except for the type specifiers (%s for the header, mixed types for the rows). Diff is unforgiving — one extra space breaks the test.

Cleanup, in two passes

You allocated two things: an array of ProcEntry*, and an array of struct dirent*. Each contains heap-allocated objects. Free the objects in a loop, then free the arrays themselves:

// Free every ProcEntry, then free the array of pointers.
for (int i = 0; i < myprocs_count; i++) {
    DestroyProcEntry(myprocs[i]);
}
free(myprocs);

// Free every dirent copy, then free the array of pointers.
for (int i = 0; i < n; i++) {
    free(dirents[i]);
}
free(dirents);

return EXIT_SUCCESS;
The cleanup invariant Every malloc/strdup/scandir in the program has a matching free on the cleanup path. If you can count one and not the other, you've got a leak. DestroyProcEntry handles three frees (struct + two strings) per call; the dirent loop handles one free per iteration.
Going deeper — error paths leak too The cleanup above runs on the success path. If scandir fails or the myprocs malloc fails, you also need to free anything allocated before the failure. In a small program, an early return EXIT_FAILURE with a few targeted frees is fine. In larger programs, you'd push everything through a single cleanup: label using goto. (Yes, goto for cleanup is idiomatic C — see the Linux kernel source.)

The complete main, in one piece

For reference, here's the full main — every section above stitched together:

int main(int argc, char **argv)
{
    const char *base_path = "/proc";
    int sort_by_comm = 0;
    int zombies_only = 0;

    int opt;
    while ((opt = getopt(argc, argv, "d:pczh")) != -1) {
        switch (opt) {
            case 'd': base_path   = optarg; break;
            case 'p': sort_by_comm = 0;     break;
            case 'c': sort_by_comm = 1;     break;
            case 'z': zombies_only = 1;     break;
            case 'h': print_usage(argv[0]); return EXIT_SUCCESS;
            default: print_usage(argv[0]); return EXIT_FAILURE;
        }
    }

    struct dirent **dirents = NULL;
    int n = scandir(base_path, &dirents, pid_filter, NULL);
    if (n < 0) { perror("scandir"); return EXIT_FAILURE; }

    ProcEntry **myprocs = malloc(sizeof(ProcEntry *) * n);
    if (myprocs == NULL) { /* free dirents and bail */ }

    int myprocs_count = 0;
    for (int i = 0; i < n; i++) {
        char stat_path[PATH_MAX];
        snprintf(stat_path, sizeof(stat_path), "%s/%s/stat",
                 base_path, dirents[i]->d_name);
        ProcEntry *e = CreateProcEntryFromFile(stat_path);
        if (e != NULL) myprocs[myprocs_count++] = e;
    }

    qsort(myprocs, myprocs_count, sizeof(ProcEntry *),
          sort_by_comm ? commSort : pidSort);

    fprintf(stdout, "%7s %7s %5s %5s %5s %7s %-25s %-20s\n",
            "PID", "PPID", "STATE", "UTIME", "STIME",
            "THREADS", "CMD", "STAT_FILE");
    for (int i = 0; i < myprocs_count; i++) {
        if (zombies_only && myprocs[i]->state != 'Z') continue;
        PrintProcEntry(myprocs[i]);
    }

    for (int i = 0; i < myprocs_count; i++) DestroyProcEntry(myprocs[i]);
    free(myprocs);
    for (int i = 0; i < n; i++) free(dirents[i]);
    free(dirents);
    return EXIT_SUCCESS;
}
