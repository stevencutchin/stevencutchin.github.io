Arrays Are Just Pointers

In C, an array name "decays" into a pointer to its first element. Understanding this unlocks pointer arithmetic and reveals how arrays truly work in memory.

An Array in Memory

When you declare int arr[5] = {10, 20, 30, 40, 50};, C allocates five consecutive int-sized slots in memory — one right after another with no gaps.

5 ints × 4 bytes each = 20 bytes of contiguous memory. Each element is exactly sizeof(int) apart.

Array Name = Pointer to First Element

Here's the critical insight: when you use the name arr in almost any expression, C automatically converts it into a pointer to the first element — &arr[0]. This is called "array decay."

int arr[5] = {10, 20, 30, 40, 50};

printf("%p\n", arr);       // 0x3000  — address of the array
printf("%p\n", &arr[0]);  // 0x3000  — same thing!

int *p = arr;              // perfectly legal — arr decays to a pointer
printf("%d\n", *p);       // 10 — the first element
The array name "arr" automatically becomes a pointer to the first element in most expressions.

Pointer Arithmetic

When you add an integer to a pointer, C doesn't just add that number of bytes — it adds that number of elements. If p is an int* and an int is 4 bytes, then p + 1 advances by 4 bytes, p + 2 by 8 bytes, and so on.

int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

printf("%d\n", *(p + 0));  // 10  — arr[0]
printf("%d\n", *(p + 1));  // 20  — arr[1]
printf("%d\n", *(p + 2));  // 30  — arr[2]
printf("%d\n", *(p + 3));  // 40  — arr[3]
Pointer arithmetic automatically scales by element size. p+1 is 4 bytes ahead for int*, 8 bytes for double*, etc.

Indexing Is Pointer Arithmetic

Here's the punchline: the bracket syntax arr[i] is literally defined as *(arr + i). The compiler translates indexing into pointer arithmetic. They are the same operation.

The Equivalence
arr[i]  ≡  *(arr + i)  ≡  *(i + arr)  ≡  i[arr]

Yes, 3[arr] is legal C and gives you arr[3]. Addition is commutative, so *(arr + 3) == *(3 + arr). This is a fun fact more than a practical technique — but it proves that indexing really is just pointer math. 
int arr[5] = {10, 20, 30, 40, 50};

// All four of these are identical:
printf("%d\n", arr[2]);         // 30
printf("%d\n", *(arr + 2));     // 30
printf("%d\n", *(2 + arr));     // 30
printf("%d\n", 2[arr]);         // 30 (please don't do this)

Arrays Passed to Functions Become Pointers

When you pass an array to a function, it completely loses its "array-ness." It decays into a pointer. The function receives only a pointer to the first element — it has no idea how long the array is. This is why C functions that take arrays almost always need a separate size parameter.

void print_array(int *arr, int size) {  // arr is just a pointer here
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);     // still works — indexing a pointer
    }
}

int nums[5] = {10, 20, 30, 40, 50};
print_array(nums, 5);  // nums decays to pointer, must pass size separately
Inside main, nums knows its size (20 bytes). Inside the function, it's just a pointer (8 bytes) — the size information is lost.

The One Difference

Arrays and pointers are almost the same, but there is one key difference: an array name is not a modifiable variable. You can't reassign it.

int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

p++;          // ✓ legal — p now points to arr[1]
arr++;        // ✗ COMPILE ERROR — can't modify an array name

p = arr + 3;  // ✓ legal — p now points to arr[3]
arr = p;      // ✗ COMPILE ERROR — arr is not assignable
Remember
The array name is a constant pointer — it always refers to the same block of memory. You can use it like a pointer, but you cannot change where it points. A pointer variable, on the other hand, can be reassigned to point anywhere.
Takeaway
Arrays in C are contiguous blocks of memory. The array name decays to a pointer to the first element. Indexing arr[i] is syntactic sugar for *(arr + i). Once passed to a function, an array is indistinguishable from a pointer — which is why you must always pass the size separately.
