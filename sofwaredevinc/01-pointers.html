What Is a Pointer?

A pointer is just a variable whose value is a memory address. That's it. Everything else is built on top of this single idea.

Variables Live in Memory

When you declare a variable like int x = 42;, the computer reserves a slot in memory for it. Every slot has two things: a value stored there, and an address — the numbered location of that slot.

Memory is a long row of numbered slots. Variable "x" lives at address 0x1008 and holds the value 42.
Analogy
Think of memory as a street of houses. Each house has a street address (the memory address) and something inside it (the value). A variable name like x is just a friendly name for one of those houses. A pointer is a slip of paper that has someone else's street address written on it.

A Pointer Stores an Address

A pointer is a variable that holds the address of another variable, rather than a regular data value. You declare a pointer with the * symbol in its type:

int  x  = 42;       // a normal int variable
int *p  = &x;      // a pointer-to-int, holding the address of x

The & operator ("address-of") gives you the memory address of a variable. So &x evaluates to something like 0x1008 — the slot where x lives. That address gets stored inside p.

Pointer p holds the address of x. Dereferencing (*p) follows the arrow to get the value 42.

The Two Key Operators

& — Address-Of

Put & in front of any variable to get its address. The result is a pointer value — a number representing where that variable sits in memory.

int x = 42;
printf("%p\n", &x);  // prints something like: 0x7ffd5e8c1008

* — Dereference

Put * in front of a pointer to follow it — to go to the address it contains and read (or write) the value stored there. This is called dereferencing.

int  x = 42;
int *p = &x;

printf("%d\n", *p);   // 42  — reads the value at the address p holds
*p = 99;              // changes x to 99 — writes through the pointer!
printf("%d\n", x);    // 99  — x itself changed
Key Concept
The * symbol has two completely different meanings depending on context. In a declaration like int *p, it means "p is a pointer." In an expression like *p = 99, it means "dereference p — follow the address."

Why Pointers Exist

Pointers are essential for three foundational reasons in C:

1. Functions can modify the caller's variables. Without pointers, C functions only receive copies of arguments. By passing a pointer, the function can reach back and modify the original variable.

void double_it(int *p) {
    *p = *p * 2;   // modifies the original variable
}

int x = 5;
double_it(&x);    // pass the address of x
// x is now 10
The function double_it receives a pointer to x. By dereferencing *p, it modifies main's variable directly.

2. Dynamic memory. Functions like malloc return pointers to newly allocated memory on the heap. This lets your program request memory at runtime — crucial for data structures whose size isn't known at compile time.

3. Efficient data handling. Instead of copying a large struct through function calls, you pass a pointer — just 4 or 8 bytes regardless of how large the original data is.

NULL — The Pointer to Nowhere

A pointer can be set to NULL (which is the value 0), meaning "I don't point to anything right now." This is useful for indicating the absence of data, but dereferencing a NULL pointer is one of the most common causes of crashes in C programs (the infamous "segmentation fault").

int *p = NULL;

if (p != NULL) {
    printf("%d\n", *p);   // safe — only dereference if not NULL
} else {
    printf("pointer is null\n");
}
Common Bug
Dereferencing a NULL pointer (*p when p == NULL) causes a segmentation fault — the operating system kills your program. Always check before dereferencing a pointer whose value you aren't certain about.

Pointer Size

On a 64-bit system, all pointers are 8 bytes (64 bits), regardless of what type they point to. An int*, a char*, and a double* are all 8 bytes — because all of them just hold a memory address, and modern systems use 64-bit addresses.

printf("sizeof(int*)    = %zu\n", sizeof(int*));     // 8
printf("sizeof(char*)   = %zu\n", sizeof(char*));    // 8
printf("sizeof(double*) = %zu\n", sizeof(double*));  // 8
Takeaway
A pointer is just a variable that holds an address. Use & to get an address, use * to follow an address. That's the foundation everything else in C memory management builds on.
