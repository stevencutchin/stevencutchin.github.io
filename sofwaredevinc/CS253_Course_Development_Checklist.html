CS 253 · Software Development in C

Course Development Checklist

For each block — are we giving them everything they need?

Every block in the course needs two things. If either is missing or incomplete, there's a gap. Apply this checklist to each block as we build it out.

Thing 1
Clear Learning Outcomes
A grouped set of outcomes defining what students walk away with.
  • Knowledge — what they will understand
  • Skills — what they will be able to do
  • Creations — what they will produce
Thing 2
Grouped Materials
Materials matched to each outcome type — nothing taught without support, nothing built without tools.
  • Materials to impart the knowledge
  • Materials to train the skills
  • Materials and tools for the things they build
Sequence: It starts with the groupings. Define the outcomes first, then build the materials to match. If you can't name the outcome, you don't build the material. If you have the outcome but no material, there's a gap.
