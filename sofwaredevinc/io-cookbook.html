I/O Cookbook

Common patterns you'll reach for repeatedly — shown alongside the mistakes that look almost right but aren't.

Terminal Output

Writing to the screen. The right function depends on how much structure you need.

printf — formatted output to stdout use when: printing values of any type with surrounding text
✓ correct
int    age  = 21;
double gpa  = 3.85;
char   name[] = "Alice";

printf("%s is %d, GPA %.2f\n",
       name, age, gpa);
✗ common mistake
double gpa = 3.85;

/* %d is for int — %f for double */
printf("GPA: %d\n", gpa);

/* missing \n — cursor stays on same line */
printf("done");

/* %s needs char* — not an int */
int x = 5;
printf("x = %s", x); // undefined behavior
Rule — Format specifier must match argument type exactly. Mismatches are undefined behavior, not just wrong output.
fprintf(stderr, …) — error messages use when: reporting errors and warnings that should not mix with normal output
✓ correct
if (n < 0) {
    fprintf(stderr,
        "Error: n must be >= 0, got %d\n",
        n);
    return 1;
}
✗ common mistake
if (n < 0) {
    /* Goes to stdout — gets lost when
       output is redirected to a file */
    printf("Error: n is %d\n", n);
    return 1;
}
Why it matters — stdout and stderr are separate streams. When a user runs ./prog > out.txt, stdout goes to the file but stderr still appears in the terminal. Errors written to stdout get silently buried in the output file.

Terminal Input

Reading from the keyboard. Match the function to whether you need a number, a word, or a whole line.

scanf — reading a single value use when: reading one number or one whitespace-delimited word
✓ correct
int n;
int ok = scanf("%d", &n);

if (ok != 1) {
    fprintf(stderr, "bad input\n");
    return 1;
}
printf("got %d\n", n);
✗ common mistake
int n;

/* Missing & — passes value not address */
scanf("%d", n);     // writes to garbage!

/* Ignoring return value */
scanf("%d", &n);   // n may be unset
printf("%d\n", n); // if user typed "abc"
Always check the return value. scanf returns the number of items matched. If the user types non-numeric input and you expected a number, the return is 0 and the variable is uninitialised.
fgets — reading a full line use when: reading a sentence, a filename, or any input that may contain spaces
✓ correct
char buf[128];
if (fgets(buf, sizeof(buf), stdin)
       == NULL) {
    fprintf(stderr, "EOF or error\n");
    return 1;
}
/* Strip trailing '\n' */
size_t len = strlen(buf);
if (len > 0 && buf[len-1] == '\n')
    buf[len-1] = '\0';
✗ common mistake
char buf[128];
fgets(buf, sizeof(buf), stdin);

/* '\n' still in buf — strcmp will fail */
if (strcmp(buf, "quit") == 0)  // never true!
    quit();

/* Or — no size limit (buffer overflow) */
scanf("%s", buf);  // no length cap
fgets always stores the '\n'. Strip it before comparing or storing strings. scanf("%s") without a length limit is a buffer overflow waiting to happen — never use it for user input.
Mixing scanf and fgets — flush the buffer first use when: reading a number with scanf, then reading a line with fgets

scanf reads digits, leaves '\n' — fgets grabs it immediately and returns empty

✓ correct — flush first
int n;
scanf("%d", &n);

/* Consume leftover '\n' (and anything else) */
int c;
while ((c = getchar()) != '\n'
        && c != EOF);

char name[64];
fgets(name, sizeof(name), stdin);
/* name now holds the next real line */
✗ common mistake — no flush
int n;
scanf("%d", &n);
/* '\n' still in buffer */

char name[64];
fgets(name, sizeof(name), stdin);
/* name == "" — grabbed the '\n' */
printf("[%s]\n", name); // prints "[]"

File Input

Opening a file and reading from it. The open/use/close pattern is non-negotiable.

Read a text file line by line use when: processing a file row by row — logs, data files, CSV
✓ correct
FILE *fp = fopen("data.txt", "r");
if (!fp) {
    perror("fopen");
    return 1;
}

char line[256];
while (fgets(line, sizeof(line), fp)
       != NULL) {
    /* process line here */
}

fclose(fp);
✗ common mistake
FILE *fp = fopen("data.txt", "r");
/* No NULL check — crash if file missing */

/* Loop on feof — reads last line twice */
while (!feof(fp)) {
    fgets(line, sizeof(line), fp);
    process(line);
}

/* Forgot fclose — data may be lost */
Never loop on feof. feof only becomes true after a read fails — so the loop runs one extra time and processes stale data. Loop on the return value of fgets instead.
Read formatted records with fscanf use when: file has a consistent record structure — e.g., "name score" per line
✓ correct
FILE *fp = fopen("scores.txt", "r");
if (!fp) { perror("fopen"); return 1; }

char name[64];
int  score;

while (fscanf(fp, "%63s %d",
              name, &score) == 2) {
    printf("%s: %d\n", name, score);
}
fclose(fp);
✗ common mistake
char name[64];
int  score;

/* Not checking exact count — may use
   uninitialised variables on bad lines */
while (fscanf(fp, "%63s %d",
              name, &score) != EOF) {
    printf("%s: %d\n", name, score);
}
Check for exactly the number you expect. != EOF passes when fscanf partially matched (returned 1 of 2) — comparing to the expected count catches that.

File Output

Writing to a file. The most important thing: always close.

Write a formatted text file use when: producing a report, log, or structured output file
✓ correct
FILE *fp = fopen("report.txt", "w");
if (!fp) {
    perror("fopen");
    return 1;
}

fprintf(fp, "Name: %s\n", name);
fprintf(fp, "Score: %d\n", score);

fclose(fp);  /* flushes buffer to disk */
✗ common mistake
/* Mode "w" silently destroys existing file */
FILE *fp = fopen("report.txt", "w");
/* ↑ wanted "a" to append, not "w" (truncate) */

fprintf(fp, "Score: %d\n", score);

/* Missing fclose — data may never reach disk */
Mode "w" truncates immediately. The old file is gone the moment fopen succeeds — not when you write. If you need to add to an existing file, use "a". If you need to preserve the original on failure, write to a temp file and rename.
Open two files safely — close both on error use when: your program needs both an input and output file open at once
✓ correct — close in / close out
FILE *in  = fopen("input.txt",  "r");
if (!in)  { perror("in");  return 1; }

FILE *out = fopen("output.txt", "w");
if (!out) {
    perror("out");
    fclose(in);   /* don't leak in! */
    return 1;
}

/* ... use in and out ... */

fclose(in);
fclose(out);
✗ common mistake — leaking in
FILE *in  = fopen("input.txt",  "r");
if (!in)  { return 1; }

FILE *out = fopen("output.txt", "w");
if (!out) {
    /* forgot fclose(in) — file handle leaked */
    return 1;
}

CSV Parsing

Reading structured data from a comma-separated file. Combines file I/O, string splitting, and type conversion.

The canonical CSV loop use when: processing a CSV file where you own the format (no quoted fields)

outer loop = fgets per line · inner loop = strtok per field

✓ correct
FILE *fp = fopen("students.csv", "r");
if (!fp) { perror("fopen"); return 1; }

char line[256];
int lineno = 0;

while (fgets(line, sizeof(line), fp)) {
    lineno++;
    /* Strip '\n' */
    size_t n = strlen(line);
    if (n > 0 && line[n-1] == '\n')
        line[n-1] = '\0';
    if (lineno == 1) continue; /* header */

    char *name  = strtok(line,  ",");
    char *major = strtok(NULL, ",");
    char *gpa_s = strtok(NULL, ",");
    if (!name || !major || !gpa_s)
        continue; /* malformed line */

    double gpa = atof(gpa_s);
    printf("%s (%s) %.2f\n",
           name, major, gpa);
}
fclose(fp);
✗ common mistakes
/* strtok on a string literal — crash */
char *line = "Alice,CS,3.85";
char *tok = strtok(line, ","); // UB

/* '\n' not stripped — strcmp breaks */
char *major = strtok(NULL, ",");
if (strcmp(major, "CS") == 0) // false!
    ...;                     // major is "CS\n"

/* Not checking tok for NULL */
double gpa = atof(strtok(NULL, ","));
/* crashes if field is missing */

Error Handling

Detecting failures early and reporting them clearly.

perror — system error messages use when: an I/O function fails and you want the OS reason
✓ correct
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) {
    /* Prints: "fopen: No such file or directory" */
    perror("fopen");
    return 1;
}
✗ common mistake
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) {
    /* Vague — no system error message */
    printf("could not open file\n");
    return 1;
    /* Also: wrote to stdout, not stderr */
}
perror prefix convention: pass the function name as the string — e.g. perror("fopen"). Output: fopen: No such file or directory. The OS message after the colon comes from the global errno variable set by the failing function.
feof / ferror — distinguishing EOF from error use when: you need to know why a read stopped after the loop exits
✓ correct
char line[256];
while (fgets(line, sizeof(line), fp)) {
    process(line);
}
/* After the loop — check why it stopped */
if (ferror(fp))
    fprintf(stderr, "read error\n");
else if (feof(fp))
    printf("finished normally\n");
✗ common mistake
/* Looping on feof — processes last
   line twice because feof only becomes
   true AFTER a read returns NULL */
while (!feof(fp)) {
    fgets(line, sizeof(line), fp);
    process(line); // double-processes last line
}

Common Mistakes — Quick Reference

The errors that appear in nearly every first pass at I/O code.

scanf & needs

Every scanf argument must be a pointer. Writing scanf("%d", x) instead of scanf("%d", &x) causes a write to garbage memory. The compiler usually warns — read the warnings.

fgets keeps \n

fgets stores the newline character at the end of the string. String comparisons like strcmp(buf, "quit") silently fail because you're comparing "quit\n" with "quit". Always strip it.

fopen NULL check

If fopen fails it returns NULL. Passing NULL to fgets or fprintf causes a segfault. Check the return value of every fopen before using the handle.

missing fclose

Data written to a file is buffered in memory. fclose flushes it to disk. Skip it and the output file may be empty or truncated — even if no error was reported during writing.

feof loop

while (!feof(fp)) runs the loop body one extra time after the real last line, because feof only becomes true after a read fails. Loop on the return value of fgets or fscanf instead.

strtok on literal

strtok writes '\0' bytes into its argument to produce tokens. String literals are read-only. Always use a writable array: char buf[] = "...", never char *p = "...".

mode "w" destroys

Opening a file with "w" truncates it to zero bytes immediately — before you write a single character. If you meant to append, use "a". There is no undo.

getchar returns int

Storing getchar()'s return in a char means EOF (−1) gets truncated, so the loop never terminates or terminates on the wrong character. Always use int c = getchar().

errors to stdout

Writing error messages with printf sends them to stdout. When stdout is redirected to a file, error messages disappear into it silently. Use fprintf(stderr, ...) for all error output.

scanf / fgets mix

scanf leaves its trailing newline in the buffer. The next fgets call picks it up immediately and returns an empty string. Flush the line after any scanf before calling fgets.

%s without length

scanf("%s", buf) with no width limit will overflow the buffer if the user types more than it holds. Use scanf("%63s", buf) for a 64-byte buffer (63 chars + null terminator).

unchecked strtok NULL

If a CSV line has fewer fields than expected, strtok returns NULL for the missing ones. Passing NULL to atof or strncpy crashes. Always check the token before using it.
