Camera Integration and Media Handling
📷 Expo Camera SDK 🎞 expo-image-picker 🔐 Permissions API 💾 MediaLibrary

The camera is one of the most powerful and user-visible hardware features on a mobile device. In this lesson you'll learn how Expo wraps the native camera hardware into a safe, cross-platform JavaScript API, how to request the permissions that make it all legal, and how to persist, display, and manage the media your app creates.

1. The Big Picture: How Camera Access Works

Unlike a web browser, a phone camera is deeply integrated with the operating system. Before any app can see a single pixel through that lens, the OS must grant explicit permission, the app must open a native camera view (or invoke the system camera), and the resulting data must be handed back as a file the app can work with. Expo provides three distinct packages that each cover a slice of this workflow.

Figure 1: Three-layer architecture — your React Native code calls Expo SDK wrappers, which in turn call native OS APIs for hardware and permissions.
Key Insight: Expo doesn't give you direct access to camera hardware. It gives you a JavaScript interface that talks to native code, which talks to the OS. This indirection is what makes cross-platform development possible — the same launchCameraAsync() call works on both iOS and Android even though the underlying hardware APIs are completely different.

2. The Three Packages You'll Use

Expo splits camera and media functionality across several focused packages rather than one monolithic library. Understanding what each package is for prevents you from reaching for the wrong tool.

Package What It Does When to Use It
expo-camera Embeds a live camera preview directly inside your UI as a React component. Gives you frame-level control — zoom, flash, focus, front/back toggle. When you need a custom camera interface baked into your app layout (think Snapchat, scanning QR codes, AR overlays).
expo-image-picker Launches the system camera or photo library as a separate modal screen. Returns when the user picks or captures media. When you just need a photo or video from the user and don't care about the capture UI. The simplest path for most apps.
expo-media-library Reads from and writes to the device's photo gallery. Lets you save images, list albums, query assets. When your app needs to save photos to the camera roll, or browse existing media.
Heads up — breaking changes: expo-camera underwent a major API redesign in Expo SDK 50. The old <Camera> component was replaced by <CameraView>. If you find tutorials using the old API, check their publication date. This course uses SDK 52.

3. Permissions — The Gate You Must Open First

Every camera and media operation is gated behind a user-granted permission. If you skip this step, your API calls will silently fail or throw errors. Permissions in Expo follow a consistent pattern across all packages.

📷
CAMERA
Required to open the camera preview or capture photos/video through expo-camera or expo-image-picker
🎤
MICROPHONE
Required separately when recording video with audio. Camera permission alone is not enough.
🖼
MEDIA LIBRARY
Required to read from or write to the photo gallery. On iOS, "Add Photos Only" is a separate, narrower permission.

The Permission Request Pattern

Every permission follows the same three-step flow:

  1. Check — ask the OS what the current permission status is without showing a dialog.
  2. Request — if not yet granted, show the system permission dialog to the user.
  3. Handle the result — act on what the user decided; never assume they said yes.
import { useState, useEffect } from 'react';
import * as ImagePicker from 'expo-image-picker';

export default function CameraExample() {
  const [hasPermission, setHasPermission] = useState(null);

  useEffect(() => {
    (async () => {
      // Step 1 + 2: request permission (checks first automatically)
      const { status } = await ImagePicker.requestCameraPermissionsAsync();
      setHasPermission(status === 'granted');
    })();
  }, []);

  // Step 3: handle result before rendering camera features
  if (hasPermission === null) return <LoadingSpinner />;
  if (hasPermission === false) return <NoPermissionMessage />;

  return <YourCameraUI />;
}
Best practice: Request permissions at the moment they're needed, not at app startup. If the user opens your app and immediately sees a camera permission dialog with no context, they often deny it. Wait until they tap "Take Photo" — then request permission. The OS will remember their choice for subsequent sessions.
Figure 2: Permission state machine. Once denied, the app cannot re-prompt — the user must go to device Settings manually.

4. expo-image-picker — The Easy Path

For the vast majority of use cases — profile photos, uploading a receipt, attaching an image to a post — expo-image-picker is the right choice. It leverages the system UI, which users already know, and handles all the complexity of camera modes, flash, and image format negotiation for you.

Installation

# Add to your Expo project
npx expo install expo-image-picker

Launching the Camera vs. the Photo Library

import * as ImagePicker from 'expo-image-picker';

// ── Option A: Launch camera to take a new photo ──────────────────────
const takePhoto = async () => {
  const result = await ImagePicker.launchCameraAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,   // show crop UI after capture
    aspect: [4, 3],          // enforce aspect ratio in crop
    quality: 0.85,           // 0–1, affects file size
  });

  if (!result.canceled) {
    const uri = result.assets[0].uri;  // local file:// path
    console.log('Captured photo at:', uri);
  }
};

// ── Option B: Pick from existing photo library ───────────────────────
const pickFromLibrary = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.All,  // photos + videos
    allowsMultipleSelection: true,  // iOS 14+ / Android
    selectionLimit: 5,
  });

  if (!result.canceled) {
    const assets = result.assets;  // array of { uri, width, height, type }
    assets.forEach(asset => console.log(asset.uri));
  }
};
The result.assets array was introduced in SDK 48. Older code checked result.uri directly. If you see tutorials using the old pattern, they need updating. Always use result.assets[0].uri.

The Asset Object

Each item in result.assets contains:

PropertyTypeDescription
uristringLocal file:// path to the image or video. Use this to display or upload the file.
widthnumberPixel width of the media.
heightnumberPixel height of the media.
type'image' | 'video'Media type.
fileNamestring?Original filename if available. May be null on some devices.
fileSizenumber?File size in bytes.
durationnumber?Video duration in milliseconds. Null for images.
base64string?Only present if base64: true was passed in options. Avoid for large files.
exifobject?EXIF metadata (GPS, exposure, etc.) if exif: true was passed.

5. expo-camera — Embedded Camera Preview

When you need the camera to live inside your UI — overlaid with your own controls, real-time filters, or information — expo-camera gives you a CameraView component that renders the live feed as a native view element.

Installation

npx expo install expo-camera

Basic CameraView with Capture

import { useRef, useState, useEffect } from 'react';
import { View, Button, Image, StyleSheet } from 'react-native';
import { CameraView, useCameraPermissions } from 'expo-camera';

export default function EmbeddedCamera() {
  const [permission, requestPermission] = useCameraPermissions();
  const [photoUri, setPhotoUri] = useState(null);
  const cameraRef = useRef(null);

  if (!permission)    return <View />;  // still loading
  if (!permission.granted) {
    return (
      <View style={styles.container}>
        <Button title="Grant Camera Access"
                onPress={requestPermission} />
      </View>
    );
  }

  const takePicture = async () => {
    if (cameraRef.current) {
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.8,
        skipProcessing: false,  // apply auto-corrections
      });
      setPhotoUri(photo.uri);
    }
  };

  return (
    <View style={styles.container}>
      <CameraView
        ref={cameraRef}
        style={styles.camera}
        facing="back"          // or "front"
        flash="auto"
      />
      {photoUri && <Image source={{ uri: photoUri }} style={styles.preview} />}
      <Button title="📷 Take Photo" onPress={takePicture} />
    </View>
  );
}
Figure 3: The CameraView component renders as a native layer. Your own JSX floats above it. The shutter button calls takePictureAsync() via a ref.
Why useRef? takePictureAsync() is an imperative command — "do this now" — rather than something you control through React state. The ref gives you a direct handle to the native camera component so you can call its methods on demand.

6. expo-media-library — Saving and Reading the Gallery

Capturing a photo only stores it in a temporary cache directory. If you want the user's photo to survive app restarts and appear in their Photos app, you need to save it to the device's permanent media library.

Installation

npx expo install expo-media-library

Saving a Photo to the Camera Roll

import * as MediaLibrary from 'expo-media-library';

const savePhoto = async (photoUri) => {
  // 1. Request permission to access / modify media library
  const { status } = await MediaLibrary.requestPermissionsAsync();
  if (status !== 'granted') {
    alert('Cannot save without photo library permission');
    return;
  }

  // 2. Save the temporary URI as a permanent asset
  const asset = await MediaLibrary.createAssetAsync(photoUri);

  // 3. Optionally organise into an album
  let album = await MediaLibrary.getAlbumAsync('MyApp Photos');
  if (album === null) {
    // First time — create the album with this asset
    await MediaLibrary.createAlbumAsync('MyApp Photos', asset, false);
  } else {
    await MediaLibrary.addAssetsToAlbumAsync([asset], album, false);
  }

  alert('Photo saved to your library! 🎉');
};
Figure 4: A captured photo lands in a temporary cache first. You must explicitly call createAssetAsync() to persist it, or POST the URI to upload it.

7. Displaying Captured Media

Once you have a uri, displaying the image is straightforward with React Native's built-in <Image> component. For videos, use expo-av's <Video> component.

📸 Displaying an Image

import { Image } from 'react-native';

<Image
  source={{ uri: photoUri }}
  style={{ width: 300, height: 200 }}
  resizeMode="cover"
/>

🎬 Playing a Video

import { Video } from 'expo-av';

<Video
  source={{ uri: videoUri }}
  style={{ width: 300, height: 200 }}
  useNativeControls
  isLooping
/>
Performance tip: Don't display full-resolution camera photos at small sizes. If you're showing thumbnails, pass quality: 0.4 in launchCameraAsync or use the width/height crop options to resize at capture time. Large images in <FlatList> will cause jank.

8. Uploading Media to a Server

A local file:// URI is only useful on the device that created it. To store media persistently or share it, you need to upload it. The standard approach uses FormData with a fetch POST request. 

const uploadPhoto = async (localUri) => {
  // Build a multipart form — the server will receive this like a file input
  const formData = new FormData();
  formData.append('photo', {
    uri: localUri,
    name: 'upload.jpg',
    type: 'image/jpeg',
  });

  const response = await fetch('https://your-api.com/photos', {
    method: 'POST',
    headers: { 'Content-Type': 'multipart/form-data' },
    body: formData,
  });

  const data = await response.json();
  console.log('Uploaded URL:', data.url);
};
Expo's FileSystem.uploadAsync() is an alternative to fetch for uploads. It supports progress callbacks, background uploads, and better error handling for large files. Worth knowing about for the final project if you're handling video.

9. Configuring app.json for Permissions

Expo's managed workflow configures native permission declarations for you — but only if you tell it what your app needs in app.json. Without these entries, your app will crash when it tries to request camera access on a production build. 

// app.json
{
  "expo": {
    "name": "My Camera App",
    "plugins": [
      [
        "expo-camera",
        {
          "cameraPermission": "Allow $(PRODUCT_NAME) to access your camera to capture photos."
        }
      ],
      [
        "expo-image-picker",
        {
          "photosPermission": "Allow $(PRODUCT_NAME) to access your photos.",
          "cameraPermission": "Allow $(PRODUCT_NAME) to open the camera."
        }
      ],
      [
        "expo-media-library",
        {
          "photosPermission": "Allow $(PRODUCT_NAME) to access your photo library.",
          "savePhotosPermission": "Allow $(PRODUCT_NAME) to save photos to your library.",
          "isAccessMediaLocationEnabled": true
        }
      ]
    ]
  }
}
Permission strings matter. Apple App Store reviewers read these strings. They must explain why your app needs access, not just what it accesses. "Access camera" will get flagged. "Capture photos for your cooking journal" will not.

10. Common Pitfalls and How to Avoid Them

PitfallSymptomFix
Permission not requested before API call API returns canceled: true or throws silently Always check/request before calling any camera API. Use the pattern from Section 3.
Missing app.json plugin config Works in Expo Go but crashes on standalone build Add all required plugin entries with permission strings to app.json.
Using old result.uri pattern undefined URI, images don't display Use result.assets[0].uri — required since SDK 48.
Trying to upload a file:// URI to a browser Broken image URL on web Always upload to a server first; file:// paths are device-local only.
Assuming photo persists between app runs Image missing after app restart Save to MediaLibrary or upload to server if persistence is required.
Using base64: true for large images App freezes, memory warnings Only use base64 for thumbnails. For uploads, use FormData with the uri.
Old <Camera> component (pre-SDK 50) Import errors, deprecated warnings Replace with <CameraView> and useCameraPermissions().
