Web Resources

This document collects the essential online resources for CS 402, organized by course theme. These are the official documentation sites, tutorials, and reference guides where you can deepen your understanding of the topics we cover. Bookmark this page — you'll return to it all semester.

A note on documentation: Official docs are your best friend. Stack Overflow answers get stale, blog posts go out of date, and AI assistants can hallucinate — but the official documentation for React Native, Expo, and the platform guidelines are maintained by the teams who build these tools. When in doubt, start there.

Core Tools — Start Here

These are the primary platforms and tools we use throughout the entire course. You'll reference these constantly regardless of which theme you're working in.

Expo

Expo — Home
https://expo.dev/
The Expo home page. Create an account here, download Expo Go, and access EAS Build and Submit services.
Expo Documentation
https://docs.expo.dev/
The complete Expo documentation — your primary reference for the development environment, SDK APIs, and build/deploy workflows. Start with the "Get Started" section if you're new.
Expo Tutorial — Build Your First App
https://docs.expo.dev/tutorial/introduction/
A hands-on walkthrough that takes you from zero to a working app with Expo Router, images, modals, and gestures. Excellent complement to the first weeks of class.
Expo — Development Workflow Overview
https://docs.expo.dev/workflow/overview/
Explains the core development loop, key terminology (Expo app, dev client, CNG), and how Expo's tools fit together. Helps you build the right mental model early.
Expo Go — Download
https://expo.dev/go
Download the Expo Go app for your iOS or Android device. This is how you run and test your projects on real hardware during development.
Expo SDK API Reference
https://docs.expo.dev/versions/latest/
Full API reference for every module in the Expo SDK — camera, location, notifications, file system, sensors, and more. The reference you'll reach for when using any Expo library.

React Native

React Native — Home
https://reactnative.dev/
The official React Native site. Overview of the framework, showcase of apps built with it, and links to documentation and community resources.
React Native Documentation
https://reactnative.dev/docs/getting-started
Official docs covering all core components (View, Text, Image, FlatList, etc.), APIs, and guides. This is the authoritative reference for React Native behavior beyond what Expo wraps.

React

React — Home
https://react.dev/
The official React documentation. React Native is built on React, so understanding the core React model (components, props, state, hooks, effects) starts here. The new docs are excellent — interactive, well-written, and example-driven.
React — Learn React
https://react.dev/learn
The guided learning path within the React docs. Covers thinking in React, describing the UI, adding interactivity, managing state, and escape hatches (refs and effects). Directly applicable to React Native.
React — API Reference
https://react.dev/reference/react
Detailed reference for every hook (useState, useEffect, useRef, useCallback, useMemo, useContext) and API in React. The "Usage" and "Troubleshooting" sections for each hook are particularly useful.

Theme 1: Foundations — Mobile Platforms & What Makes Them Unique 1

Resources for understanding mobile hardware, software architectures, platform differences, and the development environment.

Apple — Designing for iOS
https://developer.apple.com/design/human-interface-guidelines/designing-for-ios
Apple's platform-specific guidance for iOS apps. Covers the unique characteristics of the iPhone as a platform — screen sizes, interaction patterns, and what users expect.
Android Developers — Build for Android
https://developer.android.com/develop
Google's developer portal for Android. Useful for understanding Android-specific behaviors like the back button, activity lifecycle, and permission model — context for why React Native abstractions exist.
Android — Platform Architecture
https://source.android.com/docs/core/architecture
Technical overview of the Android software stack from kernel to application framework. Maps directly to the layered architecture discussion in Theme 1.
React Native — AppState API
https://reactnative.dev/docs/appstate
Documentation for React Native's AppState API, which exposes the app lifecycle events (active, background, inactive) discussed in the Foundations theme.
Expo — Device Module
https://docs.expo.dev/versions/latest/sdk/device/
Expo's Device API for querying hardware information — device type, model, OS version, memory. Useful for understanding the hardware your app runs on.

Theme 2: Building — React Native from the Ground Up 2

Resources for JavaScript, JSX, components, layout, state management, hooks, and building interactive apps.

JavaScript

MDN — JavaScript Guide
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide
Mozilla's comprehensive JavaScript guide. The definitive free resource for learning the language. Focus on the sections about grammar, control flow, functions, expressions/operators, and working with objects.
MDN — JavaScript Reference
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference
The complete JavaScript language reference. When you need to look up how Array.map() works, what the spread operator does, or the rules of destructuring — this is where you go.
The Modern JavaScript Tutorial
https://javascript.info/
A well-structured, community-maintained tutorial covering JavaScript from basics to advanced topics. Especially good for students coming from Java or C++ who need to understand JS-specific patterns like closures, promises, and async/await.

Components, Layout, and Styling

React Native — Core Components and APIs
https://reactnative.dev/docs/components-and-apis
Index of all built-in React Native components (View, Text, TextInput, Image, ScrollView, FlatList, etc.) and APIs. Each page includes props, usage examples, and platform notes.
React Native — Layout with Flexbox
https://reactnative.dev/docs/flexbox
How Flexbox layout works in React Native, including the mobile-specific defaults (column direction, no units). Essential reading for understanding how to position and size components.
React Native — Style
https://reactnative.dev/docs/style
How styling works in React Native — StyleSheet.create(), inline styles, and the differences from web CSS. Includes the style property reference for all common components.
CSS-Tricks — A Complete Guide to Flexbox
https://css-tricks.com/snippets/css/a-guide-to-flexbox/
While written for web CSS, this visual guide to Flexbox is the clearest explanation of the layout model available. The concepts transfer directly to React Native (just remember: default direction is column, not row).

State, Hooks, and Navigation

React — useState
https://react.dev/reference/react/useState
Everything about the useState hook: how it works, how React batches updates, why you shouldn't mutate state directly, and common pitfalls. Required reading for understanding reactivity in React Native.
React — useEffect
https://react.dev/reference/react/useEffect
The useEffect hook explained — dependency arrays, cleanup functions, and when to use (and not use) effects. The "You Might Not Need an Effect" section is particularly valuable.
Expo Router — Introduction
https://docs.expo.dev/router/introduction/
Expo Router provides file-based navigation for React Native apps. Covers stack navigation, tabs, modals, and deep linking — the navigation patterns discussed in Themes 2 and 3.

Theme 3: Quality — Design, Testing, and User Experience 3

Resources for UI design principles, platform guidelines, accessibility standards, and testing tools.

Design Guidelines

Apple — Human Interface Guidelines (HIG)
https://developer.apple.com/design/human-interface-guidelines/
Apple's comprehensive design guidelines. Covers touch targets, navigation patterns, visual hierarchy, typography, color, and accessibility. The standard against which iOS apps are judged.
Material Design 3
https://m3.material.io/
Google's design system for Android and cross-platform apps. Covers components, layout, color systems, typography, and interaction patterns. The Android counterpart to Apple's HIG.
Material Design 3 — Foundations
https://m3.material.io/foundations
The foundational principles of Material Design: accessibility, layout, interaction states, content design, and adaptive design. Good starting point for understanding design systems.

Accessibility

W3C — Web Content Accessibility Guidelines (WCAG) Overview
https://www.w3.org/WAI/standards-guidelines/wcag/
The international standard for digital accessibility. While written for web content, the WCAG principles (perceivable, operable, understandable, robust) and success criteria like color contrast ratios apply directly to mobile apps.
W3C — WCAG 2 at a Glance
https://www.w3.org/WAI/standards-guidelines/wcag/glance/
A quick-reference summary of the WCAG guidelines. If the full specification is overwhelming, start here for the key principles in plain language.
React Native — Accessibility
https://reactnative.dev/docs/accessibility
How to implement accessibility in React Native: accessibilityLabel, accessibilityRole, accessibilityState, and integration with VoiceOver (iOS) and TalkBack (Android).

Testing

Jest — JavaScript Testing Framework
https://jestjs.io/
The standard test runner for React and React Native projects. Documentation covers setup, matchers, mocking, snapshot testing, and async testing — everything you need for unit and integration tests.
Jest — Getting Started
https://jestjs.io/docs/getting-started
Write your first test, understand matchers, configure Jest for your project. The fastest path from zero to running tests.
React Native Testing Library (RNTL)
https://callstack.github.io/react-native-testing-library/
The recommended library for testing React Native components. Encourages testing user-visible behavior rather than implementation details. Includes render, queries, fireEvent, and userEvent APIs.
Testing Library — RNTL Introduction
https://testing-library.com/docs/react-native-testing-library/intro/
Introduction to RNTL within the broader Testing Library ecosystem. Explains the guiding philosophy: "The more your tests resemble the way your software is used, the more confidence they can give you."

Theme 4: Architecture & Engineering at Scale 4

Resources for client-server architecture, HTTP, databases, CI/CD, and the engineering practices that take apps from prototype to production.

HTTP and APIs

MDN — HTTP
https://developer.mozilla.org/en-US/docs/Web/HTTP
Comprehensive guide to HTTP — methods, status codes, headers, CORS, caching, and authentication. Understanding HTTP is essential for building the client-server communication layer.
MDN — Using the Fetch API
https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch
How to use the Fetch API to make HTTP requests — GET, POST, headers, JSON parsing, error handling. The same fetch() API works in React Native for communicating with your backend.

Databases and Persistence

Firebase Documentation
https://firebase.google.com/docs
Firebase provides real-time databases, authentication, cloud storage, and hosting. A common backend-as-a-service for mobile apps when you don't want to build your own server.
Expo — SQLite
https://docs.expo.dev/versions/latest/sdk/sqlite/
Expo's SQLite module for local on-device databases. Useful for offline-first apps and local caching — a key part of the data tier in mobile architecture.
Expo — AsyncStorage
https://docs.expo.dev/versions/latest/sdk/async-storage/
Simple key-value storage for React Native. Good for user preferences, small amounts of cached data, and app state persistence across sessions.

CI/CD and Deployment

Expo — EAS Build
https://docs.expo.dev/build/introduction/
Expo Application Services Build — compile your React Native app into native iOS and Android binaries in the cloud. The build stage of the CI/CD pipeline.
Expo — EAS Submit
https://docs.expo.dev/submit/introduction/
Submit your built app to the Apple App Store and Google Play Store directly from the command line. The deployment stage of the CI/CD pipeline.
Expo — EAS Update (Over-the-Air Updates)
https://docs.expo.dev/eas-update/introduction/
Push JavaScript updates to your users without going through app store review. Enables rapid iteration on bug fixes and small feature updates.
GitHub Actions Documentation
https://docs.github.com/en/actions
GitHub's built-in CI/CD platform. Automate linting, testing, and building on every push or pull request. Integrates well with EAS Build for a complete mobile CI/CD pipeline.
