Stylesheets & Layout
in React Native

How styling works without CSS files, why Flexbox defaults are different on mobile, and the mental model for building adaptive layouts from scratch.

01 ¬∑ The Mental Model

Styling Is JavaScript, Not CSS

React Native does not use CSS files, class names, or selectors. Instead, styles are plain JavaScript objects that you attach directly to components via the style prop. This is a fundamental departure from web development ‚Äî there is no cascade, no specificity, no external stylesheet to link.

‚úï Web CSS ‚Äî Not used in React Native
Selectors target HTML elements by tag, class, or ID. Styles cascade and inherit. Loaded from external .css files. Units include px, em, rem, %, vh.
‚úì React Native ‚Äî JavaScript objects
Style objects are created with StyleSheet.create(). Applied directly to components via the style prop. Properties are camelCase. Most dimensions are unitless numbers (density-independent pixels).
The styling pipeline: define JS objects ‚Üí attach via style prop ‚Üí React Native translates to native views.
üí°
Why StyleSheet.create()? You could pass raw objects to style, but StyleSheet.create() validates your styles at creation time (catching typos early) and lets React Native optimize by sending styles to the native side once, then referencing them by ID. It's a performance and correctness win.
02 ¬∑ Creating Stylesheets

StyleSheet.create() In Detail

A StyleSheet is created by passing an object where each key is a style name and each value is an object of style properties. Property names use camelCase (not kebab-case), and most numeric values represent density-independent pixels ‚Äî a unit that React Native auto-scales for different screen densities.

ProfileCard.js ‚Äî A complete example 
import { StyleSheet, View, Text, Image } from 'react-native';

export default function ProfileCard({ name, role, avatar }) {
  return (
    <View style={styles.card}>
      <Image source={{ uri: avatar }} style={styles.avatar} />
      <View style={styles.info}>
        <Text style={styles.name}>{name}</Text>
        <Text style={styles.role}>{role}</Text>
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    flexDirection: 'row',       // children side-by-side
    alignItems: 'center',        // vertically centered
    padding: 16,                 // 16 dp all sides
    backgroundColor: '#ffffff',
    borderRadius: 12,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.08,
    shadowRadius: 8,
    elevation: 3,               // Android shadow
  },
  avatar: {
    width: 56,
    height: 56,
    borderRadius: 28,            // half of width = circle
  },
  info: {
    marginLeft: 14,
    flex: 1,                     // take remaining space
  },
  name: {
    fontSize: 18,
    fontWeight: '700',
    color: '#1B3A5C',
  },
  role: {
    fontSize: 14,
    color: '#888',
    marginTop: 2,
  },
});

Key Differences from Web CSS

class="card" style={styles.card} No selectors ‚Äî styles are applied directly to components

‚ö†Ô∏è
No style inheritance across components. In web CSS, setting color: navy on a parent makes all text inside inherit that color. In React Native, each <Text> component must be styled independently. The only exception: nested <Text> inside another <Text> does inherit text styles.
03 ¬∑ Layout

Flexbox ‚Äî The Only Layout System

React Native uses Flexbox exclusively for layout. Every <View> is a flex container by default. The critical difference from web CSS: the default flexDirection is 'column', not 'row'. This reflects mobile's natural vertical scrolling behavior.

Default flexDirection is 'column' in React Native (vertical stacking), unlike web CSS which defaults to 'row'.

The Core Flexbox Properties

PropertyControlsCommon Values
flexDirection The main axis direction 'column' (default) ¬∑ 'row'
justifyContent Distribution along the main axis 'flex-start' ¬∑ 'center' ¬∑ 'space-between' ¬∑ 'space-around' ¬∑ 'space-evenly'
alignItems Alignment along the cross axis 'stretch' (default) ¬∑ 'flex-start' ¬∑ 'center' ¬∑ 'flex-end'
flex How much space a child claims relative to siblings 1 ¬∑ 2 ¬∑ 0
gap Spacing between children (no margins needed) Any number (density-independent pixels)
justifyContent distributes children along the main axis. alignItems positions them along the cross axis.
04 ¬∑ The flex Property

Proportional Space with flex

The flex property controls how children divide available space. A child with flex: 2 takes twice the space of a sibling with flex: 1. This is the primary tool for building adaptive layouts that work across different screen sizes.

The flex property distributes remaining space proportionally. Combine with fixed widths for hybrid layouts.
05 ¬∑ Dynamic Styling

Changing & Combining Styles

Static styles get you far, but real apps need styles that respond to state ‚Äî a button that changes color when pressed, a card that highlights when selected, a layout that adapts to screen orientation. React Native provides several patterns for this.

Combining Multiple Styles with Arrays

The style prop accepts an array of style objects. Later styles override earlier ones, giving you a clean base + override pattern:

Combining styles with arrays 
// Base style + conditional override
<View style={[
  styles.button,                            // always applied
  isActive && styles.buttonActive,          // applied when isActive is true
  isDisabled && styles.buttonDisabled,      // applied when isDisabled
]}>

// Inline overrides (use sparingly)
<Text style={[styles.label, { color: textColor }]}>

// The array pattern in the stylesheet:
const styles = StyleSheet.create({
  button: {
    paddingVertical: 12,
    paddingHorizontal: 24,
    borderRadius: 8,
    backgroundColor: '#2E5D8A',
  },
  buttonActive: {
    backgroundColor: '#1B3A5C',     // overrides the base
    transform: [{ scale: 0.97 }],   // slight press effect
  },
  buttonDisabled: {
    backgroundColor: '#ccc',
    opacity: 0.6,
  },
});

State-Driven Layouts

Because styles are JavaScript, you can compute them based on state, props, or device dimensions. The useWindowDimensions hook is particularly useful for responsive layouts:

Responsive layout with useWindowDimensions 
import { useWindowDimensions } from 'react-native';

export default function AdaptiveLayout() {
  const { width } = useWindowDimensions();
  const isWide = width > 600;       // tablet or landscape

  return (
    <View style={{
      flexDirection: isWide ? 'row' : 'column',
      padding: isWide ? 24 : 16,
    }}>
      <View style={{ flex: isWide ? 1 : 0, width: isWide ? undefined : '100%' }}>
        <Text>Sidebar / List</Text>
      </View>
      <View style={{ flex: 2 }}>
        <Text>Content Area</Text>
      </View>
    </View>
  );
}
Same component, different layouts ‚Äî driven by useWindowDimensions() and conditional flexDirection.
‚úÖ
Best practice: Keep most styles in StyleSheet.create() and use arrays or conditional logic for the parts that change. Avoid putting entire style objects inline ‚Äî it creates new objects on every render, which defeats React Native's optimization and hurts performance.
06 ¬∑ Spacing

Padding, Margin, and Gap

Spacing in React Native follows the CSS box model: padding is internal spacing (inside the border), margin is external spacing (outside the border), and gap adds uniform spacing between flex children without needing margin on each child.

The box model: margin ‚Üí border ‚Üí padding ‚Üí content. Use gap on the parent instead of margin on each child for uniform spacing.
Spacing shorthand properties 
const styles = StyleSheet.create({
  container: {
    padding: 16,                   // all four sides
    paddingHorizontal: 20,          // left + right
    paddingVertical: 12,            // top + bottom
    paddingTop: 24,                 // single side
    gap: 12,                        // between flex children
    rowGap: 8,                      // gap between rows only
    columnGap: 16,                  // gap between columns only
  },
  card: {
    margin: 8,                      // outside space, all sides
    marginBottom: 16,               // extra space below
  },
});
07 ¬∑ Quick Reference

Common Layout Patterns

Most mobile screens are built from a handful of patterns. Here's how to achieve each one:

PatternKey Styles
Center everything flex: 1, justifyContent: 'center', alignItems: 'center'
Horizontal row of items flexDirection: 'row', alignItems: 'center', gap: 12
Header + scrollable content Header: fixed height. Content: flex: 1 wrapping a <ScrollView>
Sticky footer Parent: flex: 1. Content: flex: 1. Footer: fixed height at bottom
Two equal columns Parent: flexDirection: 'row'. Both children: flex: 1
Full-screen background flex: 1, backgroundColor or <ImageBackground> with flex: 1
Circle avatar width: 56, height: 56, borderRadius: 28 (half of width)
üîë
The golden rule: If a component should fill available space, give it flex: 1. If it should have a fixed size, give it explicit width and height. Most layout bugs come from forgetting to set flex: 1 on the outermost container ‚Äî without it, your app renders at zero height.
