The Four-Layer Course Roadmap

The Whole Bicycle

Before we look at any single part, here is the complete picture of what it means to build mobile applications with React Native. Every app you create this semester exists in all four of these layers simultaneously. Understanding React Native means understanding all of them and how they connect.

The key insight: these layers are not topics you learn one at a time. They are perspectives you develop simultaneously. When you write const [count, setCount] = useState(0); that single line has a story in every layer — syntax that declares it (Code), a reconciler that re-renders when it changes (Runtime), a device that displays the result on a physical screen (Platform), and testing tools that verify it behaves correctly (Ecosystem).

React Native Code

What you write

The syntax and semantics of React Native development. JavaScript and TypeScript fundamentals, JSX for declarative UI, functional components, hooks (useState, useEffect, useRef), props and one-way data flow, StyleSheet for styling, and event handlers for interactivity. This is the vocabulary and grammar of every app you build.

React Native Runtime

What happens

The full lifecycle from JSX to pixels. How the React reconciler diffs the virtual DOM, how state changes trigger re-renders, how the JavaScript thread communicates with the native thread, how FlatList virtualizes long lists, how navigation manages screen stacks in memory, and how side effects synchronize with the render cycle.

Mobile Platform

Where it lives

The actual device and operating system your app runs on. Battery life constraining background work, limited memory forcing efficient rendering, touch-based input requiring generous tap targets, sensors (GPS, camera, accelerometer) enabling unique capabilities, network variability demanding offline resilience, and iOS/Android conventions shaping user expectations.

Dev Ecosystem

How you work

The tools and practices of professional mobile development. Expo CLI for bootstrapping and running apps, Jest and React Native Testing Library for quality assurance, EAS for cloud builds and deployment, React DevTools for inspecting components, git for version control, and CI/CD pipelines that automate the path from code to app store.

How the Layers Connect

The four layers are not independent — they explain each other. Understanding one layer deepens your understanding of the others. Here's how they talk to each other:

RN Code ↔ RN Runtime

The code defines what you can express. The runtime shows what actually happens when you do. You can't understand useState (code) without understanding the re-render cycle (runtime). You can't understand why FlatList is preferred over ScrollView (code choice) without understanding virtualization (runtime optimization). When your list stutters, that's the runtime telling you something about how your code maps to actual rendering work.

RN Runtime ↔ Mobile Platform

The runtime is an abstraction of the platform. React Native components map to real UIKit views on iOS and Android Views on Android. The JavaScript thread and native thread are scheduled by the OS. When the OS moves your app to the background, it affects your running JavaScript. When the device is low on memory, the OS can kill your app — that's the platform enforcing boundaries your runtime must respect.

RN Code ↔ Mobile Platform

React Native was designed to bridge code and platform naturally. Platform-specific components let your code feel native on each OS. Responsive layouts adapt your code to different screen sizes. Permissions APIs let your code request access to hardware. When you call expo-camera, that single line of code reaches through to physical hardware — the directness is what makes mobile development powerful and what makes understanding the platform essential.

Dev Ecosystem ↔ Everything

Development tools exist to help you work effectively across all three other layers. Expo translates your code into a runnable app on real devices. Jest tests your component logic (code) to verify it triggers the right behavior (runtime). EAS Build compiles for both platforms simultaneously. React DevTools lets you inspect the runtime. Git preserves your work. CI/CD automates the full pipeline from code to app store. The ecosystem wraps around every other layer.

Where the Four Themes Live

The four course themes — Foundations, Building, Quality, and Architecture — aren't separate from the four layers. Each theme emphasizes certain layers while touching all of them. This is the map between what we teach and where it lives.
