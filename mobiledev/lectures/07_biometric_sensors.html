Biometric Sensors: Fingerprint & Face Recognition

★ TRICORDER

Identifies crew members via bio-signs. Medical Tricorder scans vital signs and biological patterns.

★ YOUR PHONE

Touch ID reads your fingerprint in milliseconds. Face ID maps 30,000 infrared dots onto your face to build a 3D model. Both authenticate you without passwords. The Taptic Engine provides physical feedback to confirm actions.

1. Fingerprint Sensors

1.1 Capacitive Fingerprint Readers

The original Touch ID (and most Android fingerprint readers) uses a capacitive sensor. The sensor surface contains a dense grid of tiny capacitor plates. When you place your finger on the sensor, the ridges of your fingerprint are closer to the plates than the valleys. Ridges create higher capacitance, valleys create lower capacitance. This capacitance map — at high enough resolution — is a unique image of your fingerprint.

Figure 7.1 — Capacitive fingerprint sensor. The ridge/valley pattern of a fingerprint creates a unique capacitance map across the sensor array.

1.2 Ultrasonic Fingerprint Readers

Newer Android flagships (notably Samsung Galaxy) use ultrasonic fingerprint sensors embedded under the display. Instead of measuring capacitance, the sensor emits ultrasonic sound pulses that bounce off your finger. Ridges and valleys reflect the sound differently, creating a 3D depth map of the fingerprint. This works even with wet or oily fingers, where capacitive sensors struggle.

1.3 Optical In-Display Readers

Some phones use an optical sensor under the OLED display. The screen itself illuminates your finger, and a camera sensor underneath captures the reflected light pattern. This is the simplest and cheapest in-display solution but is less secure than ultrasonic, as it captures a 2D image rather than a 3D depth map.

2. Face ID: Structured Light

2.1 How It Works

Apple’s Face ID (and similar Android implementations) uses a TrueDepth camera system consisting of three components working together:

Figure 7.2 — Face ID projects 30,000 infrared dots onto your face. The IR camera reads the deformation pattern to build a 3D depth map. Works in complete darkness.

The dot projector emits a grid of 30,000 infrared dots onto your face. The IR camera captures the resulting pattern. Because the dots land on a 3D surface (your face), their positions are distorted compared to a flat plane. The system analyzes this distortion to construct a mathematical 3D depth map of your face. A flood illuminator provides even IR lighting so the system works in complete darkness.

The 3D depth map is what makes Face ID secure against photos and masks. A printed photo of someone’s face reflects the dots on a flat plane — the depth pattern is completely wrong. The system also adapts over time, learning gradual changes like growing a beard or wearing glasses.

Security Architecture: Biometric data (fingerprint minutiae, face depth map) is stored in a dedicated Secure Enclave — a physically isolated processor on the SoC that the main CPU and operating system cannot access. Apps never see your actual fingerprint or face data. Instead, they ask the OS “is this the device owner?” and receive a yes/no answer. This is the security model behind the expo-local-authentication API.

3. The Haptic Engine

While not a sensor in the traditional sense, the Taptic Engine (Apple) or haptic motor (Android) is the feedback complement to biometric sensors. When Face ID or Touch ID confirms your identity, the haptic engine provides a subtle physical “click” that confirms the action without visual feedback. Modern haptic engines use linear resonant actuators that can produce a wide range of precisely controlled vibration patterns — from the sharp confirmation tap of successful authentication to the gentle pulse of a notification.

4. As a Developer: Expo LocalAuthentication

import * as LocalAuthentication from 'expo-local-authentication';

// Check if biometric auth is available
const compatible = await LocalAuthentication.hasHardwareAsync();
const enrolled = await LocalAuthentication.isEnrolledAsync();

if (compatible && enrolled) {
  const result = await LocalAuthentication.authenticateAsync({
    promptMessage: 'Authenticate to view your account',
    fallbackLabel: 'Use passcode',
  });
  
  if (result.success) {
    // User authenticated - show protected content
  }
}
Platform Abstraction: Notice that the Expo API does not distinguish between Face ID and Touch ID. You call authenticateAsync() and the OS uses whatever biometric hardware is available. This is the right level of abstraction for most apps — you care about whether the user authenticated, not how.

6. The Full Tricorder Picture

This completes our tour of the mobile sensor suite. From the MEMS accelerometer that detects gravity’s pull, to the GPS receiver listening to satellites 20,200 km overhead, to the IR dot projector mapping the contours of your face — every one of these sensors converts some physical phenomenon into data that your app can use.

The Star Trek Tricorder was a storytelling device — it did whatever the plot needed. Your phone is a real engineering achievement, bound by real physics, real battery constraints, and real permission models. Understanding how each sensor works — not just its API — is what separates a developer who uses sensors from a developer who uses them well.
