Essential elements

One token → one owner

Identity (contract + token_id) is permanent; ownerOf(t) resolves to a single address at any instant. The token is a pointer, not a tag.

The move is the sale

The owner pointer changes only at a transfer, and that state change is the receipt — direction gives buyer vs. seller, no external proof needed.

Every owner is a query key

Each address the token passes through is a handle into the ETH graph. The trace hands step 2 its entry tickets — the wallets to pivot and link.
