Create the linking graph

Start from the initial set of addresses and run a breadth-first search over their Ethereum transactions. Each hop pulls in the counterparties of the current frontier; where branches meet, addresses land in one component. The output is a connectivity graph.

Essential elements

The seed set is the frontier

The initial addresses are where BFS begins — the hop 0 frontier every later ring grows out from.

Each hop adds counterparties

Every address in the frontier contributes its direct ETH transaction partners. Those become the next frontier; each edge is a transaction between two addresses.

Meeting branches connect

When two expansions reach the same address, their seeds join one component. That shared reachability is the connectivity the graph records.
