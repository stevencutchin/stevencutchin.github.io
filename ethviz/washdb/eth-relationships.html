ETHEREUM FUNDAMENTALS

Address, ETH, transaction, network

The Ethereum market is three things and the graph they form: an address holds a balance of ETH, a transaction moves ETH from one address to another, and the full history of transactions forms the Ethereum transaction network — a directed graph over every address.

One transaction · exactly two endpoints · ETH one way · both balances change · recorded forever.

Every transaction ever made is an edge. Together they are one directed graph over all addresses — the Ethereum transaction network.

The four entities

Address

A wallet — the actor, and the same entity that owns NFTs. It holds an ETH balance and sends or receives transactions. Free to create; the chain records addresses, not people.

ETH

The currency. Fungible — an amount with no identity, unlike a token. It exists only as balances on addresses, and moving it is what a transaction does.

Transaction

One transfer of ETH between exactly two addresses: (from, to, value, t). Public, timestamped, permanent — sending one requires both parties' involvement, so it evidences a real relationship.

Network

All transactions together: a directed graph with addresses as nodes and transactions as edges, spanning Ethereum's entire history. This is the graph the linking graph is built from.

The relationships

ADDRESS ⟶ ETH · one balance
Every address holds exactly one ETH balance. It is a running sum — the net of every transaction the address has ever sent and received.
TRANSACTION ⟶ ADDRESS · exactly two
A transaction connects exactly one sender to one receiver, moving one amount at one time. It is the atomic unit of the market.
ADDRESS ⟶ TRANSACTION · many
An address takes part in any number of transactions, incoming and outgoing. Its full history is its neighborhood in the graph.
TRANSACTIONS ⟶ NETWORK · edges of one graph
The set of all transactions is the network: addresses as vertices, transactions as directed edges. Any two addresses are related exactly as the paths between them say.
