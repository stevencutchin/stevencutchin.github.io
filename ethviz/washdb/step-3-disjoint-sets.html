THE METHOD · TOŠIĆ ET AL. · ALG. 203

Cluster a token's addresses into disjoint sets

Run once per NFT ownership trace. Seed candidate sets from the token's transfers, merge sets that share a member, then merge further using links from the linking graph. The output is that token's disjoint sets — the groups used to flag its wash trades.

transfer (weight 0) — seeds a set trade (weight > 0) link — from the linking graph disjoint set — output
Input
01

One token's trace, plus the linking graph

Two inputs: G_nft — a single token's ownership trace from step 1 (transfers and priced trades, time-ordered) — and G_ln — the linking graph from step 2.

Setup
02

Seed one set per transfer

Walk the trace. For every transfer (weight 0), add the pair {source, target} as a set. Priced trades seed nothing; an address that never touches a transfer never enters a set.

Process
03

Merge until nothing more can merge

  1. Merge shared members. Fuse any two sets with a common address; repeat until the sets are disjoint.
  2. Merge by link. Fuse any two sets when a member of one is linked to a member of the other in G_ln. Repeat to a fixed point.
The link merges only sets that transfers already seeded — it recruits no new address. The intermediary on the ETH path stays inside the linking graph, never a set member.
Output
04

The token's disjoint sets — then flag its trades

The result is a list of disjoint sets for this token. A priced trade is flagged when its buyer and seller both sit in one set.

Scoped to this token. The next token starts a fresh list of sets — separate tokens are not merged into shared sets.

Source: Tošić, Vičič & Hrovatin, "Beyond the surface: advanced wash-trading detection in decentralized NFT markets," Financial Innovation 11:86 (2025) — Algorithm 2.
