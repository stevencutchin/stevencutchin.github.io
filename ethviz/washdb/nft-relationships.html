NFT FUNDAMENTALS

Token, address, sale, collection

Four things make up an NFT marketplace, and everything else is a relationship between them: a collection contains tokens, a token points to exactly one owning address, and a sale swaps that ownership against ETH between two addresses.

Collection 1→many Token · Token →1 Address · Address →many Tokens (across collections).

Token one way, ETH the other, between two addresses. The owner pointer changing IS the sale — recorded on chain, no other receipt.

The four entities

Collection

One smart contract on Ethereum. It defines a family of tokens and holds the ownership ledger for all of them. Named by its contract_address.

Token

One unique item inside a collection — identity is contract + token_id, fixed for life. It carries a pointer to its current owner.

Address

A wallet — the actor. It owns tokens and holds ETH. Free to create, so one person can hold many addresses; the chain records addresses, not people.

Sale

A swap between two addresses: the token's owner pointer moves one way, ETH moves the other, with a price and timestamp. At 0Ξ it's a transfer — ownership moves, no payment.

The relationships

COLLECTION ⟶ TOKEN · one to many
Every token belongs to exactly one collection, permanently. A collection holds thousands of tokens.
TOKEN ⟶ ADDRESS · exactly one
At any instant a token has exactly one owner. The token is a pointer to an address, and only a sale or transfer moves it.
ADDRESS ⟶ TOKEN · many, across collections
An address can hold any number of tokens from any number of collections — and any number of addresses can belong to one person.
SALE · address ⇄ address
The only relationship that changes state: token one way, ETH the other. Every sale is a public, timestamped fact linking two addresses.
