The Graph JSON Format

One object, three keys. Every stage of the pipeline reads it and writes it back — adding fields, never reshaping.

metadata nodes edges open attributes — anything extra rides along

The top-level shape

A graph file is a single JSON object with three keys. metadata is one object describing the whole graph. nodes and edges are flat arrays of objects — one object per address, one object per transaction.

Nodes

A node needs exactly two fields to exist: index and address. index is the node's position in the array — nodes[i].index == i, always. Every other field is a column that some pipeline stage baked in, and the loader treats them all the same way.

Edges

An edge is one transaction. It needs its own index plus its two endpoints, and the endpoints are named twice on purpose: as addresses (source, target) for humans and joins, and as integers (source_index, target_index) so a renderer resolves endpoints to positions by array lookup — no address hashing at draw time.

Endpoint resolution is an array index, not a lookup: nodes[edge.source_index].

The open-attribute rule

Beyond the required core, a node or edge object may carry any additional keys, and different objects in the same array may carry different keys. The loader discovers the full attribute set by scanning, builds a column for every key it finds, and fills gaps with nulls. Nothing needs to be declared anywhere.

Three objects with different keys become five columns. Missing values are null.

Extra attributes come along for free. A column costs nothing until a visual channel binds to it — color, size, filter, tooltip. Unbound columns sit in memory, appear in the attribute selector, and are otherwise ignored. Adding a field to your data never requires touching the loader.
Constraint on values. Attribute values are JSON scalars (number, string, boolean, null) or arrays of scalars. Nested objects are not columns and are rejected with a warning — flatten them into dotted keys ("stats.degree") before writing.

The minimal valid file

Everything the loader strictly requires — two nodes, one edge, empty metadata:

{
  "metadata": {},
  "nodes": [
    { "index": 0, "address": "0xaaaa…0001" },
    { "index": 1, "address": "0xaaaa…0002" }
  ],
  "edges": [
    { "index": 0,
      "source": "0xaaaa…0001", "target": "0xaaaa…0002",
      "source_index": 0, "target_index": 1 }
  ]
}

Every richer file is this file plus columns. See example_graph.json for a two-component graph exercising positions, groups, bridges, seeds, and ride-along attributes.
