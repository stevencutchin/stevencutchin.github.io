STEP3

Cluster into Disjoint Sets

Turn one token's transfers into disjoint sets of addresses — each set the candidate footprint of a single hand — by seeding a pair from every transfer and merging until no two sets touch.

runs once per token · everything below is scoped to token T

INPUT

Two structures arrive

The token's trace — the ordered ownership sequence of token T: every movement of the token between addresses, each marked as a transfer (zero value) or a trade (priced), in time order.

The linking graph — an undirected graph over addresses. An edge between two addresses records that they are connected through Ethereum transaction activity.

transfer (0 ETH) trade (priced) linking edge candidate set

SETUP

Every transfer seeds a pair

Walk token T's trace and take each transfer edge. Its two endpoints — sender and receiver — become a 2-set: the smallest possible candidate, born directly from one zero-value movement of the token. Four transfers in the trace produce four seeds.

PROCESS

Merge until disjoint

Two union rules run over the seed sets, repeating until a full pass changes nothing. Order between the rules is free — the fixed point is the same.

RULE 1

Shared member

Two sets that contain the same address merge into one. Address A appears in both {A,B} and {A,E} — one address, one hand, one set.

RULE 2

Linkability

Two sets merge when a member of one and a member of the other are connected in the linking graph. The linking edge B — D joins {A,B,E} and {C,D}. The rule merges the sets that exist; the sets keep only the addresses they already hold.

A further pass applies both rules again and finds nothing to merge: no two sets share an address, and no linking edge crosses between them. The process is at its fixed point.

OUTPUT

Disjoint sets for token T

The result is a family of disjoint address sets, each the candidate footprint of one hand behind token T's transfers. Every address in a set traces back to a transfer of T, directly or through the merges; every set is separated from every other set both by membership and by the linking graph.

These sets are the step's deliverable: for token T, the family { {A,B,C,D,E}, {F,G} }, carried forward with the token identifier attached.
